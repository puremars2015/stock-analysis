# 股市資料分析網站內容規劃

## 零、網站實作技術選擇

用python flask框架開發網站本體
用tailwind作為主要的css框架
用sql server作為主要的資料庫
用docker作為這個網站的容器,方便開發

## 一、網站主題

本網站主題為「股市資料分析平台」，主要提供台股投資人快速查看大盤、個股、期貨、法人籌碼、公司公告與基本面資訊，並透過圖表、指標與資料整合，協助使用者進行市場觀察與投資研究。

網站定位不是下單交易平台，而是「資料查詢、視覺化分析、投資研究輔助平台」。

## 二、主要資料來源

| 資料來源           | 主要用途                        | 可應用內容                    |
| -------------- | --------------------------- | ------------------------ |
| FinMind        | 台股、期貨、財報、月營收、法人買賣、融資融券、即時資料 | K 線圖、法人籌碼、技術指標、基本面資料     |
| 公開資訊觀測站 MOPS   | 公司公告、重大訊息、財報、月營收            | 公司基本面、重大事件提醒、財報查詢        |
| 台灣期貨交易所 TAIFEX | 期貨、選擇權、三大法人、未平倉資料           | 台指期、多空籌碼、選擇權 Put/Call 分析 |
| Yahoo Finance  | 國際股市、美股、ETF、匯率、商品行情         | 全球市場、美股科技股、美元指數、國際參考行情   |
| Alpha Vantage  | 美股個股、ETF、國際股價、指數               | 美股 K 線、技術指標、國際市場比較            |

FinMind 提供超過 50 種金融資料集，包含台股日資料、5 秒資料、財報、資產負債表、現金流量表、月營收、法人買賣、期貨與選擇權資料等，且資料每天更新；但 GitHub 說明也提到其資料主要供教育與非商業用途，並有 API request 限制，設計系統時需納入快取與排程更新機制。([GitHub][1])

MOPS 的主要功能是提供投資人查詢上市、上櫃、興櫃與公開發行公司的公開資訊，適合放入公司公告、重大訊息、財務報表與月營收查詢模組。([台灣證券交易所][2])

TAIFEX 可作為台指期、選擇權與法人期貨籌碼資料來源，例如三大法人期貨多空、未平倉餘額、大額交易人未沖銷部位等，適合用於判斷期貨市場多空結構。([台灣期貨交易所][3])

Yahoo Finance 可提供歷史價格、股利、拆股與國際市場資料；若使用 yfinance，需注意它是非 Yahoo 官方授權工具，且其說明強調資料使用需遵守 Yahoo 的使用條款，偏向研究與個人用途。([Yahoo幫助中心][4])

Alpha Vantage 提供美股個股、ETF 與國際股價指數資料，包含即時報價、歷史 K 線、技術指標（如 RSI、MACD）等，支援 CSV 與 JSON 格式輸出，適合用於美股個股技術分析與國際市場比較。每日 API 呼叫次數有限制，免費版每秒最多 5 次，每日 500 次，設計系統時需納入快取機制。([Alpha Vantage][6])

| 外資台指期未平倉多空淨額       | 判斷外資期貨方向 |

TAIFEX 三大法人資料包含自營商、投信、外資及陸資等分類，並提供交易口數、契約金額與未平倉餘額等欄位，適合做成法人多空儀表板。([台灣期貨交易所][5])

# 六、資料處理流程

## 1. 資料擷取

系統定期從 FinMind、MOPS、TAIFEX、Yahoo Finance 抓取資料。

資料類型：

| 頻率      | 資料             |
| ------- | -------------- |
| 即時 / 盤中 | 大盤、個股、期貨價格     |
| 每日收盤後   | 日 K、法人買賣超、融資融券 |
| 每日定時    | 重大訊息、公告        |
| 每月      | 月營收            |
| 每季      | 財務報表           |
| 每年      | 股利、年報資料        |

---

## 2. 資料清洗

處理內容：

| 項目     | 說明                  |
| ------ | ------------------- |
| 股票代號統一 | 統一 2330、台積電、TSMC 對應 |
| 日期格式統一 | 使用 YYYY-MM-DD       |
| 缺漏值處理  | 避免圖表錯誤              |
| 單位轉換   | 元、千元、億元             |
| 重複資料排除 | 避免重複寫入              |
| 資料來源標記 | 保留資料來源與更新時間         |

---

# 十一、風險與注意事項

| 項目     | 說明                           |
| ------ | ---------------------------- |
| 資料授權   | 不同資料來源有不同授權限制，商業化前需確認授權      |
| 即時資料延遲 | 免費資料源可能不是完全即時                |
| API 限流 | FinMind 有 request 限制，需使用快取   |
| 爬蟲風險   | MOPS、TAIFEX 若無正式 API，需注意網站規範 |
| 投資風險   | 網站應註明資料僅供參考，不構成投資建議          |
| 資料正確性  | 需保留資料來源、更新時間與錯誤處理機制          |
| 系統效能   | K 線與即時報價需避免頻繁查詢資料庫           |
| 使用者體驗  | 首頁資訊多，需避免畫面過度擁擠              |

# 十三、總結

本網站第一階段應以「大盤 + 台積電 + 法人籌碼 + 個股查詢」為核心，先建立可用的資料分析儀表板。第二階段再加入期貨、選擇權、MOPS 公告、基本面與選股器。第三階段可擴充 AI 摘要、策略回測、通知提醒與個人化自選股。

> 首頁做成「台股市場儀表板」，個股頁做成「單一股票研究中心」，法人與期貨頁做成「多空籌碼觀察工具」，再透過 AI 摘要降低使用者解讀資料的門檻。
